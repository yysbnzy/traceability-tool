## 一、工具概述

### 1.1 工具定位

双向溯源工具是一款专业的**测试用例与需求双向追溯分析工具**，用于：
- 建立测试用例与需求文档的双向关联映射
- 自动生成多维度溯源报告
- 量化测试覆盖率，识别遗漏需求
- 支持用例/需求ID的前缀/后缀动态拼接

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| **双文档导入** | 用例文档（必填）+ 需求文档（可选） |
| **Sheet页选择** | 支持Excel多Sheet页选择 |
| **ID拼接** | 用例/需求ID可添加前缀/后缀（固定文本或从列读取） |
| **智能分隔** | 支持逗号、分号、顿号、斜杠、换行等多种分隔符 |
| **前缀过滤** | 按用例/需求ID前缀过滤显示 |
| **7维度输出** | 合并版、逗号版、输入源、异常分析、需求缺失、统计汇总、数据校验 |
| **覆盖率计算** | 导入需求文档后自动计算需求覆盖率 |

### 1.3 系统要求

- **操作系统**：Windows 10/11
- **运行方式**：双击EXE，无需安装Python
- **文件格式**：Excel 2010+ (.xlsx)
- **体积**：约34MB（单文件）

### 3.2 区域详解

#### 区域1：用例文档配置（必填）

| 控件 | 说明 | 默认值 |
|------|------|--------|
| **文件路径** | 选择包含用例ID和需求ID的Excel文件 | - |
| **Sheet页** | 选择Excel中的工作表 | 第一个Sheet |
| **用例ID列** | 用例ID所在的Excel列 | A |
| **需求ID列** | 关联需求ID所在的Excel列 | B |

> **截图位置**：用例文档配置区域特写，标注4个控件

#### 区域2：用例ID拼接配置（可选）

| 控件 | 说明 | 来源选项 |
|------|------|----------|
| **前缀来源** | 如何获取前缀 | 固定文本 / 用例文档列 / 需求文档列 |
| **前缀值** | 前缀内容 | 文本输入框 或 列选择下拉 |
| **后缀来源** | 如何获取后缀 | 固定文本 / 用例文档列 / 需求文档列 |
| **后缀值** | 后缀内容 | 文本输入框 或 列选择下拉 |

**拼接公式**：`最终用例ID = 前缀 + 原始用例ID + 后缀`

> **重要**：拼接仅作用于**双向溯源表**（合并版+逗号版），其他表使用原始ID

> **截图位置**：用例ID拼接配置区域，分别展示"固定文本"和"用例文档列"两种来源的界面状态

#### 区域3：需求ID拼接配置（可选）

与用例ID拼接逻辑完全一致，作用于需求ID。

#### 区域4：需求文档配置（可选但建议）

| 控件 | 说明 | 默认值 |
|------|------|--------|
| **文件路径** | 独立的需求清单Excel文件 | - |
| **Sheet页** | 选择需求文档工作表 | 第一个Sheet |
| **需求ID列** | 需求ID所在的列 | A |

**作用**：
- 计算"需求覆盖率"
- 识别"需求缺失用例"
- 为拼接配置中的"需求文档列"来源提供数据

> **截图位置**：需求文档配置区域

#### 区域5：溯源过滤配置（可选）

| 控件 | 说明 |
|------|------|
| **用例ID过滤** | 只分析/显示此前缀开头的用例（匹配原始ID） |
| **需求ID过滤** | 只分析/显示此前缀开头的需求（匹配原始ID） |

> **注意**：过滤匹配**原始ID**（不含前缀/后缀），仅影响双向溯源表显示，不影响异常分析

> **截图位置**：过滤配置区域

## 四、操作步骤详解

### 4.1 场景一：基础溯源分析（仅用例文档）

**适用场景**：手里只有一份用例文档，想建立用例与需求的关联关系

**步骤1：选择用例文档**
1. 点击用例文档配置区域的 **"浏览"** 按钮
2. 在文件选择对话框中，选择Excel文件（.xlsx）
3. 选择后，Sheet页下拉框会自动列出所有Sheet

> **截图位置**：文件选择对话框 + 选择后的界面状态

**步骤2：确认列映射**
- 检查用例ID列和需求ID列是否正确（默认A、B）
- 如果Excel结构不同，从下拉框选择正确的列

> **截图位置**：列映射下拉框展开状态

**步骤3：开始分析**
1. 点击底部 **"🚀 开始溯源分析"** 按钮
2. 弹出保存对话框，选择输出位置
3. 等待分析完成，提示"溯源分析已完成！"

> **截图位置**：保存对话框 + 完成提示弹窗

**输出**：5个Sheet（双向溯源表-合并版、双向溯源表-逗号版、输入源、异常分析、统计汇总）

---

### 4.2 场景二：完整覆盖率分析（用例+需求文档）

**适用场景**：评估测试用例对需求的覆盖程度，找出遗漏

**步骤1：配置用例文档**（同场景一）

**步骤2：配置需求文档**
1. 点击需求文档配置区域的 **"浏览"** 按钮
2. 选择独立的需求清单Excel文件
3. 选择正确的Sheet页和需求ID列

> **截图位置**：需求文档配置完成后的界面状态

**步骤3：开始分析**
- 点击 **"开始溯源分析"**，选择保存位置

**输出**：7个Sheet完整报告，重点关注 **"统计汇总"** Sheet中的"需求覆盖率"

**适用场景**：用例ID前缀需要根据Excel中的"项目代号"列动态变化

| A列-用例ID | B列-需求ID | C列-项目代号 |

| TC-001 | REQ-001 | PROJ_A |

- `TC-001` → `PROJ_A_TC-001`
- `TC-002` → `PROJ_B_TC-002`

## 六、常见问题

### Q1：过滤后双向溯源表为空？

**原因**：过滤条件包含隐藏字符，或过滤条件匹配的是原始ID而非拼接后的ID。

**解决**：
1. 检查过滤条件是否包含换行、零宽空格等隐藏字符
2. 确认过滤条件用的是**原始ID前缀**（不含拼接的前缀/后缀）

> **截图位置**：过滤条件输入框特写 + 过滤后的空表截图

### Q2：需求覆盖率显示为0%？

**原因**：未导入需求文档，或需求文档列配置错误。

**解决**：
1. 导入独立的需求文档
2. 确认需求文档中需求ID列和Sheet页配置正确

> **截图位置**：需求文档配置正确 vs 错误的对比截图

### Q3：双向溯源表ID和其他表不一致？

**原因**：这是设计意图——双向溯源表显示拼接后的ID，其他表保留原始ID。

**说明**：
- 双向溯源表（合并版+逗号版）：显示拼接后的ID（如 `PROJ_A_TC-001`）
- 输入源/异常分析/需求缺失：显示原始ID（如 `TC-001`）

> **截图位置**：双向溯源表 vs 输入源的ID对比截图

### Q4：孤儿用例判定错误？

**说明**：孤儿用例指该用例ID对应的需求ID列解析后为空。

**检查**：
- 该用例行的需求ID列是否确实没有内容？
- 需求ID是否因分隔符问题被过滤掉了？

### Q5：双击EXE无反应或闪退？

**可能原因**：
- Windows安全软件拦截 → 添加信任或关闭实时防护
- 文件路径含特殊字符 → 将EXE移到纯英文路径（如 `D:\Tools\`）
- 缺少系统运行库 → 安装 VC++ Redistributable
