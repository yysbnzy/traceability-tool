# 测试用例与需求溯源工具 v3.1 - 使用说明文档

## 📋 工具概述

**测试用例与需求溯源工具 v3.1** 是一款专业的测试用例与需求双向追溯分析工具。通过导入测试用例文档（可选需求文档），自动建立用例与需求的双向关联关系，生成包含统计汇总的多维度溯源报告，帮助测试团队快速评估测试覆盖率和识别遗漏。

**核心定位**：解决测试用例与需求文档之间的追溯断层问题，量化测试覆盖率。

---

## 🎯 v3.1 核心功能

### 1. 双文档导入机制
- **用例文档（必填）**：包含用例ID及关联需求的Excel文件
- **需求文档（可选）**：独立的需求清单文档，用于计算覆盖率缺口

### 2. 智能列映射
- 用例ID列、需求ID列均可自由指定（A-Z列下拉选择）
- **默认配置**：用例ID列=A，需求ID列=B

### 3. Sheet页选择（v3.1新增）
- 支持选择Excel中的任意Sheet页进行分析
- 加载文件后自动列出所有Sheet名称

### 4. 动态ID拼接（v3.1重点功能）
前缀/后缀来源支持三种模式：
- **固定文本**：手动输入固定字符串
- **用例文档列**：动态取指定列的值作为前缀/后缀
- **需求文档列**：动态取需求文档指定列的值

### 5. 智能分隔符解析
支持中文逗号 `，`、英文逗号 `,`、分号 `;`、句号 `。`、斜杠 `/`、空格、换行等多种分隔形式自动识别。

### 6. 过滤配置（v3.1新增）
- **用例ID过滤**：只分析指定前缀开头的用例
- **需求ID过滤**：只分析指定前缀开头的需求

### 7. 六维度输出报告
- **Sheet1 - 双向溯源表**：用例→需求、需求→用例双向聚合
- **Sheet2 - 输入源**：原始数据完整副本
- **Sheet3 - 异常分析**：孤儿用例、孤儿需求、被过滤项识别
- **Sheet4 - 需求缺失用例**：需求文档有但用例未覆盖的需求
- **Sheet5 - 统计汇总**：覆盖率百分比等关键指标
- **Sheet6 - 数据校验**：输入源总数与输出条目数一致性校验

## 📊 配置项详细说明

### 1. 用例文档配置（必填）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 文件路径 | - | 选择包含用例ID和需求ID的Excel文件 |
| Sheet页 | Sheet1 | 加载后自动列出所有Sheet，可选择任意页 |
| 用例ID列 | **A** | 下拉选择用例ID所在的Excel列（A-Z） |
| 需求ID列 | **B** | 下拉选择需求ID所在的Excel列（A-Z） |

> ⚠️ **注意**：请根据实际Excel结构调整列映射。

### 2. 用例ID拼接配置（可选）

用于对输出的用例ID进行格式化加工。

| 配置项 | 来源选项 | 说明 |
|--------|----------|------|
| 前缀来源 | 固定文本 / 用例文档列 / 需求文档列 | 如何获取前缀内容 |
| 前缀值 | 文本输入框 或 列选择下拉 | 根据来源类型动态切换控件 |
| 后缀来源 | 固定文本 / 用例文档列 / 需求文档列 | 如何获取后缀内容 |
| 后缀值 | 文本输入框 或 列选择下拉 | 根据来源类型动态切换控件 |

**来源类型说明**：
- **固定文本**：手动输入任意字符串（如 `TEST_`、`_V1.0`）
- **用例文档列**：从用例文档的指定列取值（如取C列作为项目代号前缀）
- **需求文档列**：从需求文档的指定列取值（需先上传需求文档）

**拼接公式**：
```
最终用例ID = 前缀 + 原始用例ID + 后缀
```

### 3. 需求ID拼接配置（可选）

与用例ID拼接逻辑完全一致，作用于输出的需求ID。

### 4. 需求文档配置（可选但建议填写）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 文件路径 | - | 独立的需求清单Excel文件 |
| Sheet页 | Sheet1 | 可选择需求文档中的任意Sheet页 |
| 需求ID列 | **A** | 需求文档中需求ID所在的列 |

**作用**：
- 计算"需求覆盖率"（已覆盖需求数 / 需求文档总需求数）
- 识别"需求缺失用例"（需求文档中有但用例未关联的需求）
- 为拼接配置中的"需求文档列"来源提供数据

> 💡 **建议**：如需完整的覆盖率分析，请导入独立的需求文档。

### 5. 溯源过滤配置（可选）

| 配置项 | 说明 |
|--------|------|
| 用例ID过滤 | 只显示/分析此前缀开头的用例ID |
| 需求ID过滤 | 只显示/分析此前缀开头的需求ID |

> 过滤仅影响双向溯源表显示，异常分析仍基于完整数据。

## 📥 输入格式要求

### 用例文档格式

Excel文件中，**用例ID列**和**需求ID列**根据界面配置指定（默认A列和B列）。

#### 支持的数据形式

**形式一：一行多需求（推荐）**

| A列-用例ID | B列-需求ID |
|-----------|-----------|
| TC-001 | REQ-001, REQ-002 |
| TC-002 | REQ-002/REQ-003 |

**形式二：多行单需求**

| A列-用例ID | B列-需求ID |
|-----------|-----------|
| TC-001 | REQ-001 |
| TC-001 | REQ-002 |
| TC-002 | REQ-002 |

> 两种形式可以混用，工具会自动聚合同一个用例ID下的所有需求。

#### 需求ID分隔符支持

工具自动识别以下分隔符：
- 中文逗号：`，`
- 英文逗号：`,`
- 分号：`;`
- 顿号：`、`
- 斜杠：`/`
- 句号：`。`
- 空格 / 换行

### 需求文档格式（可选）

仅需要一列：需求ID（默认A列）

| A列-需求ID |
|-----------|
| REQ-001 |
| REQ-002 |
| REQ-003 |

## 📤 输出报告详解

### Sheet1 - 双向溯源表

展示用例与需求的双向聚合关系（过滤后）：

| A列：用例ID | B列：关联需求ID |
|-------------|----------------|
| TC-001 | REQ-001, REQ-002 |
| TC-002 | REQ-002, REQ-003 |

| D列：需求ID | E列：关联用例ID |
|-------------|----------------|
| REQ-001 | TC-001, TC-003 |
| REQ-002 | TC-001, TC-002 |

> 已去重、排序，空关联项不显示。

### Sheet2 - 输入源

原始用例ID列和需求ID列的完整副本，用于数据核对。

### Sheet3 - 异常分析

自动识别三类异常：

| 异常类型 | 含义 | 处理建议 |
|----------|------|----------|
| 孤儿用例 | 用例ID存在但需求ID列解析后为空 | 补充需求关联或确认用例必要性 |
| 需求被过滤 | 用例存在但关联需求不符合过滤条件 | 检查过滤配置或需求ID格式 |
| 孤儿需求 | 需求ID在用例中被引用但无有效用例关联 | 检查用例ID是否为空或格式错误 |

> 🐛 **v3.1修复**：异常分析中的关联需求ID现在显示原始值（如 `SWRD-MCUSecurity-0007`），不再错误拼接版本前缀。

### Sheet4 - 需求缺失用例

当导入独立需求文档时，显示需求文档中存在但用例文档未覆盖的需求：

| 需求ID | 状态 | 说明 |
|--------|------|------|
| REQ-005 | 未覆盖 | 该需求在需求文档中存在，但未在测试用例中体现 |

### Sheet5 - 统计汇总

| 统计项 | 示例值 | 说明 |
|--------|--------|------|
| 总用例数 | 50 | 用例文档中用例ID非空的总数 |
| 双向关联用例数 | 45 | 有关联需求的用例数 |
| 孤儿用例数 | 5 | 未关联任何需求的用例数 |
| 用例中引用需求数 | 30 | 用例文档中引用的需求ID总数 |
| 双向关联需求数 | 28 | 有关联用例的需求数 |
| 孤儿需求数 | 2 | 有用例ID为空但引用的需求数 |
| 需求文档总需求数 | 35 | 需求文档中的需求总数 |
| 需求缺失用例数 | 7 | 需求文档中有但用例未覆盖的需求数 |
| **需求覆盖率** | **80.0%** | 已覆盖需求数/需求文档总需求数 |

> 需求覆盖率是衡量测试完整性的核心指标。

### Sheet6 - 数据校验

自动检查数据完整性：
- 双向溯源表条目数 + 异常分析表条目数 = 输入源去重总数
- 如果数据不匹配，会弹窗提示错误，但**文件仍然会生成**，方便排查问题。

## 💡 典型使用场景与配置示例

### 场景1：基础溯源分析（仅用例文档）

**需求**：手里只有一份用例文档，想看看用例和需求关联关系。

**操作**：
1. 选择用例文档
2. 选择正确的Sheet页
3. 确认用例ID列和需求ID列映射正确（默认A/B）
4. 点击"开始溯源分析"

**输出**：Sheet1~3（双向溯源 + 输入源 + 异常分析）

---

### 场景2：完整覆盖率分析（用例+需求文档）

**需求**：评估测试用例对需求的覆盖程度，找出遗漏。

**操作**：
1. 选择用例文档，配置Sheet页和列映射
2. 选择需求文档，配置Sheet页和列映射
3. 点击"开始溯源分析"

**输出**：6个Sheet完整报告，重点关注Sheet5的"需求覆盖率"

---

### 场景3：添加项目代号前缀

**需求**：用例ID需要统一加上项目代号前缀。

**配置**：
```
用例ID拼接配置：
  前缀来源: 固定文本
  前缀值: PROJ_A_
  后缀来源: 固定文本
  后缀值: （留空）
```

**效果**：`TC-001` → `PROJ_A_TC-001`

---

### 场景4：从Excel列动态取前缀

**需求**：用例ID前缀需要根据Excel中的"项目代号"列动态变化。

**假设Excel结构**：

| B列-需求ID | C列-项目代号 | A列-用例ID |
|-----------|-------------|-----------|
| REQ-001 | PROJ_A | TC-001 |
| REQ-002 | PROJ_B | TC-002 |

**配置**：
```
用例ID拼接配置：
  前缀来源: 用例文档列
  前缀值: C
```

**效果**：
- `TC-001` → `PROJ_A_TC-001`
- `TC-002` → `PROJ_B_TC-002`

---

### 场景5：使用过滤功能聚焦分析

**需求**：只想分析版本v7.10的用例及其关联需求。

**配置**：
```
用例ID过滤: v7.10#
需求ID过滤: V1.0#
```

**效果**：双向溯源表只显示匹配的用例和需求，异常分析中显示被过滤的条目。

## ⚠️ 常见问题与解决方案

### Q1: 提示"您选择了'需求文档列'作为来源，但未上传需求文档"

**原因**：在拼接配置中选择了"需求文档列"作为前缀/后缀来源，但没有上传需求文档。

**解决**：
- 方案A：上传需求文档
- 方案B：将来源改为"固定文本"或"用例文档列"

---

### Q2: 分析结果为空或数据不对

**检查清单**：
1. 用例ID列和需求ID列是否配置正确？
2. Sheet页是否选择正确？
3. Excel文件中对应列是否确实包含数据？
4. 数据是否从第2行开始？（第1行被识别为表头跳过）
5. 需求ID分隔符是否在支持范围内？

---

### Q3: 需求覆盖率显示为0%

**原因**：没有导入需求文档，或需求文档列配置错误。

**解决**：
1. 导入独立的需求文档
2. 确认需求文档中需求ID列和Sheet页的配置正确

---

### Q4: 孤儿用例判定错误

**说明**：孤儿用例指该用例ID对应的需求ID列解析后为空（不是"需求ID为空"的意思）。

**检查**：
- 该用例行的需求ID列是否确实没有内容？
- 需求ID是否因分隔符问题被过滤掉了？

---

### Q5: 双击EXE无反应或闪退

**可能原因**：
- Windows 安全软件拦截 → 添加信任或关闭实时防护
- 文件路径含特殊字符 → 将EXE移到纯英文路径（如 `D:\Tools\`）
- 缺少系统运行库 → 安装 VC++ Redistributable
