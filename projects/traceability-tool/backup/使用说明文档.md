# 测试用例与需求溯源工具 - 使用说明文档

## 📋 工具概述

**测试用例与需求溯源工具**是一款专业的双向追溯分析工具，用于根据Excel表格中的**用例ID**和**需求ID**列，自动生成双向溯源矩阵，帮助测试团队快速建立用例与需求之间的关联关系，确保测试覆盖率的可追溯性。

---

## 🎯 核心功能

### 1. 双向溯源矩阵生成
- **用例 → 需求**：聚合每个用例关联的所有需求ID
- **需求 → 用例**：聚合每个需求关联的所有用例ID
- **原始数据保留**：保留输入数据的完整副本

### 2. 智能数据处理
- **智能分隔符解析**：支持中文逗号 `，`、英文逗号 `,`、分号 `;`、句号 `。`、空格、换行等多种分隔形式
- **自动去重排序**：每个用例/需求只出现一次，关联项去重并按字母排序
- **多行数据聚合**：支持一个用例ID分多行、每行一个需求ID的输入格式

### 3. 高级功能（v2.0+）
- **双文档导入**：分别导入用例文档和需求文档
- **前缀过滤**：支持用例ID和需求ID的前缀识别与过滤
- **ID拼接**：支持自定义前缀/后缀拼接规则
- **异常检测**：自动识别孤儿用例、孤儿需求、前缀过滤异常
- **需求缺失分析**：检测需求文档中存在但无用例关联的需求

## 📦 安装与运行环境

### 系统要求

## 🚀 使用方式

### 方式一：GUI图形界面（推荐）

#### 启动工具

**Windows用户：**
- 双击运行 `gui.py` 或打包后的 `.exe` 文件
- 或在命令行中执行：
```bash
python gui.py
```

**Linux/macOS用户：**
```bash
python3 gui.py
```

#### 界面操作步骤

1. **选择用例文件**
   - 点击 **"📂 浏览"** 按钮
   - 选择包含用例ID和需求ID的Excel文件（.xlsx格式）

2. **配置列映射（如需要）**
   - 用例ID列：下拉选择包含用例ID的列（默认：第A列）
   - 需求ID列：下拉选择包含需求ID的列（默认：第B列）

3. **配置ID拼接规则（可选）**
   - 用例前缀/后缀：为输出的用例ID添加统一前缀或后缀
   - 需求前缀/后缀：为输出的需求ID添加统一前缀或后缀

4. **导入需求文档（可选）**
   - 如需检测需求覆盖率，可额外导入需求文档
   - 选择需求ID所在的列

5. **开始分析**
   - 点击 **"开始分析"** 按钮
   - 等待处理完成

6. **查看结果**
   - 输出文件自动保存在原文件同目录下
   - 文件名格式：`原文件名_output.xlsx`

---

### 方式二：命令行模式

#### 基础用法

```bash
python traceability.py <输入Excel路径> <输出Excel路径>
```

#### 示例

```bash
# 基础分析
python traceability.py input.xlsx output.xlsx

# 带前缀过滤
python traceability.py input.xlsx output.xlsx --case-prefix TC --req-prefix REQ

# 完整参数
python traceability.py input.xlsx output.xlsx \\
    --case-prefix TC \\
    --req-prefix REQ \\
    --case-prefix-out TEST_ \\
    --case-suffix _V1 \\
    --req-prefix-out REQ_ \\
    --req-suffix _2024
```

#### 命令行参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `input.xlsx` | 输入Excel文件路径 | `测试用例.xlsx` |
| `output.xlsx` | 输出Excel文件路径 | `溯源分析结果.xlsx` |
| `--case-prefix` | 用例ID前缀过滤 | `TC` |
| `--req-prefix` | 需求ID前缀过滤 | `REQ` |
| `--case-prefix-out` | 输出用例ID前缀 | `TEST_` |
| `--case-suffix` | 输出用例ID后缀 | `_V1` |
| `--req-prefix-out` | 输出需求ID前缀 | `REQ_` |
| `--req-suffix` | 输出需求ID后缀 | `_2024` |

## 📊 输入格式要求

### 基本格式

Excel文件**第一列**为 `用例ID`，**第二列**为 `需求ID`（可在GUI中调整列映射）。

### 支持的输入形式

#### 形式一：一行多需求（推荐）
一个用例ID一行，需求ID列中用分隔符分隔多个需求：

| 用例ID | 需求ID |
|--------|--------|
| TC-001 | REQ-001, REQ-002 |
| TC-002 | REQ-002，REQ-003 |
| TC-003 | REQ-001; REQ-003 |

#### 形式二：多行单需求
一个用例ID分多行，每行一个需求ID（工具会自动聚合）：

| 用例ID | 需求ID |
|--------|--------|
| TC-001 | REQ-001 |
| TC-001 | REQ-002 |
| TC-002 | REQ-002 |
| TC-002 | REQ-003 |

> **提示**：两种形式可以混用，工具会自动识别并聚合。

## 📈 输出格式说明

### Sheet1 - 双向溯源表（主要输出）

| 区域 | 列 | 内容说明 |
|------|-----|----------|
| 原始数据区 | A列 | 原始用例ID |
| | B列 | 原始需求ID（未分隔的原始文本） |
| 用例→需求 | D列 | 用例ID（去重，拼接后） |
| | E列 | 关联的所有需求ID（逗号分隔，去重排序） |
| 需求→用例 | G列 | 需求ID（去重，拼接后） |
| | H列 | 关联的所有用例ID（逗号分隔，去重排序） |

### Sheet2 - 输入源（v2.0+）

- 原始用例ID列 + 需求ID列的完整副本
- 用于数据核对和审计

### Sheet3 - 异常分析（v2.0+）

自动识别并分类以下异常情况：

| 异常类型 | 说明 | 处理建议 |
|----------|------|----------|
| **孤儿用例** | 没有关联任何需求的用例ID | 补充需求关联或确认用例必要性 |
| **孤儿需求** | 没有任何用例关联的需求ID | 补充测试用例或确认需求是否已废弃 |
| **前缀过滤异常** | 不符合前缀规则的ID | 检查ID命名规范或调整前缀配置 |

### Sheet4 - 需求缺失分析（v2.0+）

当导入独立的需求文档时，本表显示：
- 需求文档中存在但用例文档中未关联的需求ID
- 帮助识别测试覆盖率的缺口

## 💡 使用技巧

### 1. 处理大量数据
- 工具支持万级数据量的处理
- 建议在分析前检查Excel文件格式是否规范
- 避免空行和合并单元格

### 2. 前缀过滤的使用场景
- 当Excel中包含多种类型的ID时（如同时有BUG-ID和REQ-ID）
- 需要筛选特定项目或模块的用例/需求时
- 统一不同来源数据的命名规范

### 3. ID拼接功能
**适用场景：**
- 为输出的ID添加项目代号前缀：`PROJ_TC001`
- 为输出的ID添加版本后缀：`TC001_V2.0`
- 统一不同文档来源的ID格式

**示例配置：**
```
用例前缀(拼接): TEST_
用例后缀(拼接): _2024Q1
需求前缀(拼接): REQ_
需求后缀(拼接): _V1.0
```

**输出效果：**
```
用例ID: TEST_TC001_2024Q1
需求ID: REQ_001_V1.0
```

### 4. 需求覆盖率分析
1. 准备两份Excel文件：
   - 用例文档（含用例ID和需求ID关联）
   - 需求文档（含完整的需求ID列表）
2. 在GUI中分别导入两份文档
3. 查看Sheet4的"需求缺失"分析结果
4. 根据缺失列表补充测试用例

## ⚠️ 常见问题与解决方案

### Q1: 启动GUI时提示缺少tkinter
**解决方案：**
```bash
# Ubuntu/Debian
sudo apt-get install python3-tk

# CentOS/RHEL
sudo yum install python3-tkinter

# Windows
重新安装Python，勾选"tcl/tk and IDLE"选项
```

### Q2: 输出文件无法打开或格式错误
**可能原因及解决方案：**
- 输入文件不是标准.xlsx格式 → 重新保存为.xlsx格式
- 输入文件包含合并单元格 → 取消合并后再处理
- 权限问题 → 检查输出目录的写入权限

### Q3: 需求ID解析不正确
**检查要点：**
- 确认分隔符是否为工具支持的类型（逗号、分号、句号、空格、换行）
- 检查需求ID是否包含特殊字符
- 确认列映射配置是否正确

### Q4: 孤儿用例判定有误
**说明：**
- 孤儿用例指需求ID列解析后为空或仅有无效值的用例
- 检查原始数据中需求ID列是否确实为空
- 检查前缀过滤配置是否过于严格

### Q5: 大数据量处理缓慢
**优化建议：**
- 分批处理：将大文件拆分为多个小文件
- 关闭不必要的Excel进程释放内存
- 使用命令行模式而非GUI（资源占用更少）
