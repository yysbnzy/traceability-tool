# 用例需求溯源分析工具 v3.1 最终版 - 使用说明

> **工具版本**：溯源分析工具_v3_1_最终版.exe  
> **运行方式**：Windows 双击运行，无需安装 Python 环境  
> **配套模板**：用例需求溯源输入模板.xlsx

---

## 📋 工具简介

**用例需求溯源分析工具 v3.1 最终版** 是一款免安装的 Windows 桌面应用程序，用于分析测试用例与需求之间的双向追溯关系。通过导入测试用例文档（可选需求文档），自动生成多维度的溯源报告，帮助测试团队快速评估测试覆盖率、识别遗漏用例。

**适用场景**：
- 测试用例与需求的关联梳理
- 测试覆盖率统计（百分比）
- 遗漏需求/孤儿用例识别
- 多项目用例ID格式统一

## ⚙️ 配置项详解

### 区域1：用例文档配置（必填）

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| **文件路径** | 选择包含用例ID和需求ID的 Excel 文件（.xlsx/.xls） | - |
| **用例ID列** | 用例ID所在的 Excel 列（下拉选择 A-Z） | **A** |
| **需求ID列** | 需求ID所在的 Excel 列（下拉选择 A-Z） | **B** |

> ⚠️ **重要**：启动后请根据实际 Excel 文件调整列映射。配套模板的用例ID在 **A列**、需求ID在 **B列**。

---

### 区域2+3：ID 拼接配置（可选）

用于对输出的用例ID和需求ID进行格式化加工，支持动态拼接。

#### 前缀/后缀来源类型

| 来源类型 | 说明 | 使用场景 |
|----------|------|----------|
| **固定文本** | 手动输入固定字符串 | 统一添加项目代号前缀 |
| **用例文档列** | 从用例文档指定列取值 | 按项目代号动态分组 |
| **需求文档列** | 从需求文档指定列取值 | 按需求版本号动态标记 |

#### 拼接公式

```
最终ID = 前缀 + 原始ID + 后缀
```

**示例**：
- 原始用例ID：`SyTC-MCU_Security-0002`
- 前缀固定文本：`PROJ_A_`
- 后缀固定文本：`_V1.0`
- 最终输出：`PROJ_A_SyTC-MCU_Security-0002_V1.0`

---

### 区域4：需求文档配置（可选但强烈建议）

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| **文件路径** | 独立的需求清单 Excel 文件 | - |
| **需求ID列** | 需求文档中需求ID所在的列 | **A** |

**导入需求文档后，工具将额外提供**：
- ✅ **需求覆盖率**百分比（核心指标）
- ✅ **需求缺失用例**清单（需求文档有但用例未覆盖）
- ✅ 支持"需求文档列"作为拼接来源

> 💡 **建议**：如需完整的覆盖率分析，请务必导入独立的需求文档。

## 📥 输入数据格式

### 用例文档格式

工具支持标准 Excel 格式，表头占第1行，数据从第2行开始。

#### 配套模板示例结构

| A列：用例ID | B列：需求ID | ... |
|-------------|------------|-----|
| v7.10#SyTC-MCU_Security-0002 | | |
| v7.10#SyTC-MCU_Security-0006 | SWRD-CyberSecurity-008, SWRD-CyberSecurity-022 | |
| v7.10#SyTC-MCU_Security-0007 | SWRD-CyberSecurity-022, SWRD-CyberSecurity-023 | |

#### 填写规则

1. **第一列填写用例ID，第二列填写需求ID**（根据列映射调整）
2. **一个用例对应多个需求**时，用顿号 `、`、逗号 `,`、分号 `;` 或 **换行** 分隔
3. **同一个用例ID可分多行填写**，工具会自动聚合
4. 用例ID和需求ID支持任意命名规范（如 `SyTC-xxx`、`SWRD-xxx`）

#### 多行单需求示例（自动聚合）

| A列：用例ID | B列：需求ID |
|-------------|------------|
| SyTC-001 | REQ-001 |
| SyTC-001 | REQ-002 |
| SyTC-002 | REQ-002 |

工具会自动聚合成：`SyTC-001` → `REQ-001, REQ-002`

---

### 需求文档格式（可选）

仅需要一列需求ID（默认A列）：

| A列：需求ID |
|-----------|
| SWRD-CyberSecurity-008 |
| SWRD-CyberSecurity-022 |
| SWRD-CyberSecurity-023 |

## 📤 输出报告说明

分析完成后，工具会自动弹出保存对话框，生成 `.xlsx` 报告文件。

### Sheet1 - 双向溯源表

展示用例与需求的双向聚合关系：

| A列：用例ID | B列：关联需求ID |
|-------------|----------------|
| v7.10#SyTC-MCU_Security-0006 | SWRD-CyberSecurity-008, SWRD-CyberSecurity-022 |
| v7.10#SyTC-MCU_Security-0007 | SWRD-CyberSecurity-022, SWRD-CyberSecurity-023 |

| D列：需求ID | E列：关联用例ID |
|-------------|----------------|
| SWRD-CyberSecurity-008 | v7.10#SyTC-MCU_Security-0006 |
| SWRD-CyberSecurity-022 | v7.10#SyTC-MCU_Security-0006, v7.10#SyTC-MCU_Security-0007 |

> 数据已去重、排序，空关联项不显示。

---

### Sheet2 - 输入源

原始用例ID和需求ID列的完整副本，用于核对原始数据。

---

### Sheet3 - 异常分析

自动识别两类异常数据：

| 异常类型 | 含义 | 处理建议 |
|----------|------|----------|
| **孤儿用例** | 用例ID存在，但未关联任何需求 | 补充需求关联或确认用例必要性 |
| **孤儿需求** | 需求ID被引用但无有效用例关联 | 检查用例ID是否为空或格式错误 |

---

### Sheet4 - 需求缺失用例

当导入独立需求文档时，显示需求文档中存在但用例未覆盖的需求：

| 需求ID | 状态 | 说明 |
|--------|------|------|
| SWRD-CyberSecurity-999 | 未覆盖 | 该需求在需求文档中存在，但未在测试用例中体现 |

---

### Sheet5 - 统计汇总 ⭐

| 统计项 | 示例值 | 说明 |
|--------|--------|------|
| 总用例数 | 993 | 用例文档中用例ID非空的总数 |
| 双向关联用例数 | 950 | 有关联需求的用例数 |
| 孤儿用例数 | 43 | 未关联任何需求的用例数 |
| 用例中引用需求数 | 120 | 用例文档中引用的需求ID总数 |
| 双向关联需求数 | 115 | 有关联用例的需求数 |
| 孤儿需求数 | 5 | 有用例ID为空但引用的需求数 |
| 需求文档总需求数 | 150 | 需求文档中的需求总数 |
| 需求缺失用例数 | 35 | 需求文档中有但用例未覆盖的需求数 |
| **需求覆盖率** | **76.7%** | 已覆盖需求数 ÷ 需求文档总需求数 |

> **需求覆盖率**是衡量测试完整性的核心指标，建议保持在 90% 以上。

## 💡 典型使用场景

### 场景1：基础溯源分析（仅用例文档）

**适用**：快速梳理现有用例与需求的关联关系。

**操作步骤**：
1. 运行 `溯源分析工具_v3_1_最终版.exe`
2. 点击"📂 浏览"选择用例文档
3. **调整列映射**为用例ID列=A、需求ID列=B（根据模板）
4. 点击"🚀 开始溯源分析"
5. 选择保存位置，完成

**输出**：Sheet1~3（双向溯源 + 输入源 + 异常分析）

---

### 场景2：完整覆盖率评估（用例+需求文档）

**适用**：评估测试用例对需求的覆盖程度，找出遗漏。

**操作步骤**：
1. 配置用例文档（列映射调整正确）
2. 点击需求文档区域的"📂 浏览"选择需求清单
3. 调整需求ID列映射（默认A列）
4. 点击"🚀 开始溯源分析"
5. 查看 Sheet5 的"需求覆盖率"

**重点关注**：Sheet4（需求缺失用例）+ Sheet5（覆盖率%）

---

### 场景3：统一添加项目前缀

**适用**：不同项目/版本的用例需要区分标识。

**配置示例**：
```
用例ID拼接配置：
  前缀来源: 固定文本
  前缀值: PROJ_A_
  后缀来源: 固定文本
  后缀值: （留空）
```

**效果**：`SyTC-001` → `PROJ_A_SyTC-001`

---

### 场景4：从Excel列动态取前缀

**适用**：用例ID前缀需要根据文档中的"项目版本"列自动变化。

**假设用例文档结构**：

| A列：用例ID | B列：需求ID | C列：项目版本 |
|-------------|------------|--------------|
| SyTC-001 | REQ-001 | v7.10 |
| SyTC-002 | REQ-002 | v7.11 |

**配置**：
```
用例ID拼接配置：
  前缀来源: 用例文档列
  前缀值: C
```

**效果**：
- `SyTC-001` → `v7.10SyTC-001`
- `SyTC-002` → `v7.11SyTC-002`

## ⚠️ 常见问题

### Q1: 双击EXE无反应或闪退

**可能原因**：
- Windows 安全软件拦截 → 添加信任或关闭实时防护
- 文件路径含特殊字符 → 将EXE移到纯英文路径（如 `D:\Tools\`）
- 缺少系统运行库 → 安装 [VC++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)

---

### Q2: 分析结果为空表

**排查步骤**：
1. 确认用例ID列和需求ID列映射**与实际Excel一致**（模板是A/B列）
2. 确认Excel数据从**第2行**开始（第1行是表头）
3. 确认需求ID分隔符在支持范围内（顿号、逗号、分号、换行）
4. 用配套模板测试，确认工具正常

---

### Q3: 需求覆盖率显示 0%

**原因**：未导入需求文档，或需求文档列配置错误。

**解决**：
- 导入独立的需求清单文档
- 确认需求文档中需求ID所在列映射正确

---

### Q4: 提示"您选择了'需求文档列'作为来源，但未上传需求文档"

**原因**：拼接配置中选择了"需求文档列"作为前缀/后缀来源，但没有上传需求文档。

**解决**：
- 方案A：上传需求文档
- 方案B：将来源改为"固定文本"或"用例文档列"

---

### Q5: 孤儿用例判定有误

**说明**：孤儿用例指该用例行的需求ID列解析后为空，不是"需求ID不存在"的意思。

**检查**：
- 该用例行的需求ID列（B列）是否确实没有内容？
- 需求ID是否因格式问题被过滤？
